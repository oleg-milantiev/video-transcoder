# Grafana Dashboard Update - Log Monitoring with Action Tracking

## Overview
Обновленный dashboard `/grafana/dashboard/log.json` теперь полностью отражает архитектуру логирования с поддержкой нового поля `action` и предоставляет детальную аналитику динамики вызовов.

## Панели Dashboard

### 1. **Top Actions by Call Count** (Bar Chart)
- **Позиция**: Левая верхняя (0,0)
- **Описание**: Показывает топ 15 действий (комбинация name + action) по количеству вызовов
- **Query**: Группирует логи по name и action, подсчитывает уникальные объекты и общее количество вызовов
- **Назначение**: Быстро определить какие действия наиболее активны в системе

### 2. **Action Dynamics by Hour** (Time Series)
- **Позиция**: Правая верхняя (12,0)
- **Описание**: Временной ряд вызовов по действиям, агрегированный по часам
- **Query**: Выводит тренд активности для каждого действия (name - action) за выбранный период
- **Назначение**: Наблюдать за динамикой вызовов действий во времени, выявлять пики активности

### 3. **Error Status by Action** (Bar Chart)
- **Позиция**: Левая середина (0,8)
- **Описание**: Распределение логов по уровням (info, error, warning) для каждого действия
- **Query**: Группирует по name, action и level, показывает количество логов каждого типа
- **Назначение**: Быстро видеть какие действия имеют проблемы и какого типа

### 4. **Action Activity Heatmap (Daily)** (Heatmap)
- **Позиция**: Правая середина (12,8)
- **Описание**: Тепловая карта активности по действиям за дни
- **Query**: Показывает интенсивность каждого действия по дням (темнее = больше активности)
- **Назначение**: Выявлять паттерны активности - какие дни более нагруженные, какие действия постоянно активны

### 5. **Top Errors by Action and Object** (Table)
- **Позиция**: Левая нижняя (0,16)
- **Описание**: Таблица с топ 15 ошибками по действиям и объектам
- **Столбцы**:
  - `action_type` - название действия (name - action)
  - `object_id` - UUID объекта, над которым произошла ошибка
  - `total_calls` - всего вызовов этого действия для объекта
  - `error_count` - количество ошибок
  - `error_rate_pct` - процент ошибок
- **Назначение**: Детальный анализ проблемных объектов и действий

### 6. **Log Levels by Category** (Bar Chart)
- **Позиция**: Правая нижняя (12,16)
- **Описание**: Распределение логов по уровням (info, error, warning, debug) для каждой категории
- **Query**: Группирует по name и level, показывает stackable столбцы
- **Назначение**: Видеть общую картину логирования по категориям

### 7. **Errors / Total (Hourly)** (Time Series)
- **Позиция**: Левая очень нижняя (0,24)
- **Описание**: Двухмерный временной ряд - общее количество логов и количество ошибок по часам
- **Query**: Вычисляет error_rate в процентах
- **Назначение**: Отслеживать тренд ошибок и их долю в общем количестве логов

### 8. **Total Calls by Action** (Bar Chart)
- **Позиция**: Правая очень нижняя (12,24)
- **Описание**: Горизонтальное распределение вызовов по всем действиям
- **Query**: Простая группировка по action с подсчетом всех вызовов
- **Назначение**: Общая статистика использования всех действий

## Ключевые особенности

### ✅ Интеграция с полем `action`
- Все панели теперь используют комбинацию `name` и `action` для более детального анализа
- Возможность отслеживать не только объекты, но и конкретные операции над ними

### ✅ Временные фильтры
- Все запросы используют `$__timeFilter(created_at)` для динамических временных фильтров
- Поддерживается интерактивный выбор периода в Grafana

### ✅ Аналитика ошибок
- Три панели фокусируются на ошибках: `Error Status by Action`, `Top Errors by Action and Object`, `Errors / Total`
- Помогает быстро найти проблемные действия и объекты

### ✅ Динамика и тренды
- `Action Dynamics by Hour` и `Errors / Total (Hourly)` показывают тренды
- `Action Activity Heatmap` выявляет паттерны по дням недели

## Использование

1. **Экспорт и импорт в Grafana**:
   ```bash
   # Копируем файл в Grafana
   cp grafana/dashboard/log.json /path/to/grafana/dashboards/
   ```

2. **В UI Grafana**:
   - Перейдите в Dashboards → Import
   - Загрузите `log.json`
   - Выберите PostgreSQL datasource с UID `afhybzup80t8gd`
   - Нажмите Import

3. **Кастомизация**:
   - Используйте временной фильтр в правом верхнем углу
   - Панели можно перетаскивать, изменять размер
   - Клик по значению в таблице покажет детали

## SQL Запросы

Все запросы используют PostgreSQL и работают с таблицей `log`:

```sql
-- Основная структура таблицы
SELECT 
  created_at,      -- время создания
  name,            -- категория (video, task, user, admin)
  action,          -- действие (create, update, delete, progress, cancel, etc)
  object_id,       -- UUID объекта
  level,           -- уровень логирования (info, error, warning, debug)
  text,            -- текст сообщения
  context          -- JSON контекст
FROM log
WHERE $__timeFilter(created_at)
```

## Рекомендуемые индексы

Убедитесь что есть индексы на полях:
```sql
CREATE INDEX idx_log_name ON log(name);
CREATE INDEX idx_log_action ON log(action);
CREATE INDEX idx_log_object_id ON log(object_id);
CREATE INDEX idx_log_level ON log(level);
CREATE INDEX idx_log_created_at ON log(created_at);
```
