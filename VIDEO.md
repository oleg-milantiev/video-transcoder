## Ролики

### Введение и обзор проекта

- мотивация создания проекта:
  - 2026 год, меня сократили на работе. Рынок вакансий просел, появилось время и желание подтянуть скилы 
  - нужна была простая задача для самообразования. Опробовал руками много интересного, для меня нового:
    - DDD, SOLID
    - серьёзное покрытие phpUnit, блок выпуска фичи без пройденных тестов, stan и e2e
    - E2E headless тесты через playwright изолированные (пустая база) и prod (реальное окружение, ежедневный автотест)
    - SSE через websocket (mercure) для realtime обновлений
    - VueJS на фронте. Не совсем моя фишка, но хотелось попробовать
    - kubernetes и terraform для инфраструктуры в облаке (ещё подтянуть, оч. интересно!)
    - замороченные docker-compose окружения (dev, prod, тесты)
    - AI помощник Github Copilot в PhpStorm для ускорения разработки :like:
    - Grafana Loki для сбора и анализа логов. Мониторинг, алерты
  - постепенно проект, мне кажется, стал полезным и как продукт, а не только как тренировочная площадка
  - вот создаю видеокурс, тоже польза!
  - выложу продукт в открытый доступ с монетизацией, заработаю миллионы :)
  - каркас этого транскодера буду использовать для других проектов, заработаю ... миллиарды? :-/
- что умеет:
  - авторизация и регистрация (через гугл)
  - админка
  - загрузка больших видео
  - ограничения гибкими тарифами
  - получение мета и превью видео
  - реалтайм уведомления и обновления статуса загрузки и кодирования
  - транскодинг в пресеты
  - мониторинг и алерты
- для кого:
  - для желающих транскодировать видео (продукт в паблике)
  - для меня (самообразование, в перспективе монетизация)
  - для backend / fullstack разработчиков, которые хотят посмотреть на реальный проект с DDD, тестами, инфраструктурой и т.п.

### Архитектура и компоненты

- backend (API)
  - DDD, SOLID
    - Domain, Application, Infrastructure, Presentation, сервисы и границы
    - Domain:
      - Сущности (например, Video, Task)
      - Value Objects (например, Preset)
      - Репозитории (интерфейсы для доступа к данным)
    - Application:
      - Events (для расширения, не EDA)
      - Controller -> Query -> Handler -> Response
      - Command -> DTO -> Handler
      - Interface -> реализация (хранилище и логи)
    - Infrastructure:
      - реализация интерфейсов (репозитории, логи)
      - интеграция с внешними сервисами (S3, Redis, ffmpeg)
  - хранилище и очередь: PostgreSQL, S3 (или файлы) и Redis
